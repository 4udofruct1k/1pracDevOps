Описание
Данный проект предоставляет удобный Docker-контейнер для запуска Redis — высокопроизводительного in-memory хранилища данных, используемого для кеширования, очередей, pub/sub и многого другого.
Контейнер предназначен для быстрой интеграции в локальные и продакшн-среды, с гибкой конфигурацией через аргументы образа и переменные окружения.
Примечание: Образ основан на официальном redis:latest и предоставляет дополнительные возможности конфигурации.
Поддерживаемые аргументы образа
АргументОписаниеREDIS_PORTПорт, на котором будет запущен Redis внутри контейнера.REDIS_CONFIGПуть к вашему custom-конфигу Redis, который будет скопирован в образ.ENABLE_APPENDONLYВключает или выключает механизм журналирования (AOF).MAXMEMORYЗадаёт лимит памяти для Redis (например, 256mb). 
Пример запуска контейнера
docker run -d \ --name my-redis \ -p 6379:6379 \ -e REDIS_PORT=6379 \ -e ENABLE_APPENDONLY=yes \ -e MAXMEMORY=256mb \ my-redis-image:latest 
Настройка с помощью переменных окружения
Вы можете настроить поведение Redis, задав значения переменных окружения:
Основные параметры
REDIS_PORT
Устанавливает порт Redis внутри контейнера.
ENABLE_APPENDONLY
yes — включает AOF
no — выключает AOF
MAXMEMORY
Ограничивает доступную память (например, 512mb, 1gb).
Дополнительные параметры
REDIS_CONFIG_FILE — путь к дополнительному конфигурационному файлу.
REDIS_PASSWORD — задаёт пароль для подключения через requirepass.
